# Demo 演示脚本

## 演示目标

展示 AI 游玩日志生成系统的完整流程：拍照上传 → AI 分析 → 生成游记。

---

## 演示准备

### 1. 环境准备

```bash
# 1. 启动后端
cd backend
cp .env.example .env
python app.py
# 或: uvicorn app:app --reload --host 0.0.0.0 --port 8000

# 2. 验证后端
curl http://127.0.0.1:8000/health
# {"status":"ok","service":"AI 游玩日志生成系统","version":"0.1.0"}
```

### 2. 准备测试照片

准备 3-5 张旅行照片（可以从手机导出），最好是：
- 不同场景（景点、餐厅、街道）
- 包含 GPS 信息
- 不同时间拍摄

如果手头没有合适的照片，可以使用测试脚本生成模拟数据。

### 3. 微信开发者工具

1. 打开微信开发者工具
2. 导入项目 `miniapp/`
3. 勾选「不校验合法域名」

## 演示流程

### 第一步: 首页展示 (30 秒)

```
1. 打开小程序 → 展示首页
2. 介绍: 项目标题、核心功能 (照片解析/AI识别/自动生成游记)
3. 指出底部的后端连接状态指示器 (绿色 = 已连接)
```

**话术:**
> "这是一个 AI 游玩日志生成器。只需上传旅行照片，系统会自动读取照片的拍摄时间和 GPS 位置，再通过 AI 识别照片中的场景、活动和氛围，最终生成一篇自然流畅的中文旅行日记。"

### 第二步: 照片上传 (1 分钟)

```
1. 点击「选择照片开始」按钮
2. 从相册选择 3-5 张照片
3. 展示缩略图预览
4. 点击「上传 N 张照片」
5. 查看上传结果 (EXIF 信息: 时间、GPS 状态)
```

### 第三步: AI 分析 (30 秒)

```
1. 点击「开始 AI 分析」
2. 展示步骤进度:
   - ✓ 读取照片信息
   - ● AI 图片分析 (进行中)
   - ○ 生成游玩日志
3. 等待完成 → 自动跳转结果页
```

**话术:**
> "现在系统正在逐张分析您的照片。首先读取每张照片的 EXIF 信息，包括拍摄时间和 GPS 坐标。然后通过 MiMo 多模态模型识别照片内容——场景是什么、用户在做什么、有没有美食、氛围如何——但这些分析都是在服务器端完成的，发送给 AI 的图片已经去除了 GPS 等隐私信息。"

### 第四步: 结果展示 (1 分钟)

```
1. 展示生成的游玩日志
   - 标题
   - 日期、城市、照片数量
   - 关键词标签
   - 游玩日记正文 (分段展示)
2. 展示每张照片的 AI 分析结果
   - 拍摄时间
   - 场景类型、活动、氛围标签
   - AI 生成的一句日记描述
```

**话术:**
> "这是最终生成的游玩日志。系统按日期和城市自动整理了照片，生成了一篇中文日记。可以看到有标题、关键词标签、以及每段按时间段组织的描述。往下滑动还可以看到每张照片的单独分析结果。"

### 第五步: 总结 (30 秒)

```
1. 回顾完整流程: 上传 → 分析 → 日记
2. 介绍技术亮点:
   - EXIF 解析
   - AI 多模态识别
   - 隐私保护
   - 可替换的地点解析
3. 扩展方向: 地图打点、旅行报告、社交分享
```

## 无 MiMo API Key 的演示方案

如果暂未获得 MiMo API Key，系统会自动使用 **Mock 模式**：

```bash
# 上传测试图片
curl -X POST http://127.0.0.1:8000/upload -F "file=@test_photo.jpg"

# 分析 (Mock 模式)
curl -X POST http://127.0.0.1:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{"photo_ids": [1], "geocode": true}'

# 生成日志
curl -X POST http://127.0.0.1:8000/diary/generate \
  -H "Content-Type: application/json" \
  -d '{"photo_ids": [1]}'

# 查看日志
curl http://127.0.0.1:8000/diary/1
```

Mock 模式会返回基于哈希的模拟分析结果，每次结果可能不同，方便演示多样性。

## 演示技巧

1. **先跑通流程**: 提前上传分析好一组照片，演示时直接展示结果
2. **突出亮点**: 强调"自动"、"AI 理解"、"隐私保护"
3. **对比说明**: "以前写游记需要手动整理照片+回忆+编写，现在只需上传照片即可"
4. **真实感**: 使用真实的旅行照片效果最佳
5. **异常处理**: 可以演示一张无 GPS 的照片，展示系统如何优雅降级

## 常见问题 (FAQ)

**Q: 为什么选择微信小程序？**
A: 微信小程序无需安装、即用即走，天然适合"拍照→处理→展示"的轻量场景。

**Q: MiMo API 怎么获取？**
A: 请联系小米 AI 平台获取 API Key，或参考官方文档。本系统采用 OpenAI-compatible 接口格式。

**Q: GPS 信息会泄露吗？**
A: 不会。原图的 GPS 信息仅存储在服务器本地数据库，发送给 AI 的图片已去除所有 EXIF 元数据。

**Q: 没有 GPS 的照片能用吗？**
A: 完全可以。系统会正常分析图片内容并生成日志，只是地点显示为"未知"。

**Q: 支持哪些图片格式？**
A: JPG、PNG、HEIC、HEIF、WebP、BMP、TIFF。推荐安装 exiftool 以获得最佳 HEIC 支持。
